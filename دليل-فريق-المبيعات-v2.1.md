# 1. هيكل النظام والأدوار

## لماذا يوجد نظام أدوار؟

كل موظف في الجيم له دور مختلف ومسؤوليات مختلفة. النظام مصمم بحيث كل شخص يرى فقط ما يحتاجه لعمله، ولا يستطيع تغيير أو حذف بيانات لا تخصه. هذا يحمي البيانات ويمنع الأخطاء.

## الأدوار المتاحة في النظام:

```
┌─────────────────────────────────────────────────────────────┐
│                    هرم الصلاحيات                            │
│                                                             │
│              ┌─────────────────┐                            │
│              │   CRM Admin     │  ← أعلى صلاحية (أتف)      │
│              └────────┬────────┘                            │
│                       │                                     │
│              ┌────────▼────────┐                            │
│              │   Super Admin   │  ← صلاحية كاملة            │
│              └────────┬────────┘                            │
│                       │                                     │
│              ┌────────▼────────┐                            │
│              │     Admin       │  ← إدارة الإعدادات          │
│              └────────┬────────┘                            │
│                       │                                     │
│              ┌────────▼────────┐                            │
│              │    Manager      │  ← مدير الفريق              │
│              └────────┬────────┘                            │
│                       │                                     │

│              └─────────────────┘                            │

# 2. الدخول والتنقل

## أين؟
الصفحة الرئيسية عند فتح الرابط.

```

│                                        │
│   1. افتح رابط النظام                   │

> **ملاحظة:** أول دخول يُعطيك دور "Rep" تلقائياً. المدير يغيّر دورك من الإعدادات.

## التنقل بين الصفحات

### على الكمبيوتر (Desktop):
شريط علوي يحتوي على التبويبات حسب دورك:

```
┌──────────────────────────────────────────────────────────────────┐
│  [Logo]  Dashboard | Leads | Members | Tasks | Payments |        │
│          Attendance | Reports | History | Settings    [🔔] [أتف] │

### على الموبايل:
شريط سفلي + قائمة "المزيد":

```
┌────────────────────────────────────┐
│         المحتوى الرئيسي            │
├────────┬────────┬────────┬─────────┤
│  Home  │ Leads  │Members │  Scan  │
│   🏠   │   👤   │  👥    │  📷    │
└────────┴────────┴────────┴─────────┘
```

اضغط **"المزيد"** لرؤية: Tasks / Payments / Reports / History / Settings / Logout

---

## البحث العالمي (Global Search)

**أين:** في رأس الصفحة، مربع البحث
**لماذا:** للوصول السريع لأي عضو أو ليد بدون الدخول لأي قسم
**متى تستخدمه:** عندما تريد إيجاد شخص بسرعة بالاسم أو الرقم

**كيف:**
1. اضغط على مربع البحث في الأعلى
2. اكتب اسم العميل أو رقم هاتفه أو رقم عضويته
3. النتائج تظهر فوراً

---

## الإشعارات (🔔)

**أين:** أيقونة الجرس في رأس الصفحة
**لماذا:** تنبيهك بأشياء تحتاج اهتمام فوري
**متى تستخدمه:** عند رؤية رقم أحمر على الجرس

تنبيهات تلقائية:
- ليد لم تتواصل معه منذ 7 أيام ⚠️
- عيد ميلاد عضو اليوم 🎂
- اشتراك ينتهي خلال 3 أيام ⏰
- مهمة متأخرة عن موعدها 📋

# 4. إدارة الليدز

## أين؟
تبويب **"Leads"** في شريط التنقل العلوي

## لماذا موجود قسم الليدز؟
الليد هو أي شخص محتمل يريد الاشتراك ولم يدفع بعد. النظام يساعدك على:
- تتبع كل محادثة معه
- معرفة أين هو في رحلة الشراء
- عدم نسيان أحد
- قياس كفاءة مصادر الليدز

## متى تستخدمه؟
- عند وصول استفسار جديد من أي مصدر
- لمتابعة الليدز اليومية
- لمراجعة من لم يُتابَع

---

## مخطط رحلة الليد الكاملة:

```
┌──────────────────────────────────────────────────────────────────┐
│                    رحلة الليد من البداية للنهاية                  │
└──────────────────────────────────────────────────────────────────┘

جاء استفسار جديد
        │
        ▼
┌───────────────┐
│  أضف ليد جديد │
│  (New Stage)  │
└───────┬───────┘
        │
        ▼
 تواصل معه ← سجّل Interaction
        │
        ├─── لم يرد ──────────────────► Follow Up (متابعة)
        │                                      │
        │                              حاول مرة ثانية
        │                                      │
        │                         ┌────────────┤
        │                         │            │
        │                      اهتم        لم يهتم
        │                         │            │
        │                         ▼            ▼
        │                     Trial Stage   Lost ❌
        │
        ├─── مهتم ────────────────► حدد موعد تجربة
        │                              │
        │                    جاء للتجربة؟
        │                    ┌─────────┴─────────┐
        │                    │                   │
        │                   نعم                  لا
        │                    │                   │
        │                    ▼                   ▼
        │             هل قرر الاشتراك؟       Follow Up
        │             ┌─────┴─────┐
        │             │           │
        │            نعم          لا

## التبويبات في صفحة Leads:

```
┌──────┬─────────┬───────────┬─────────┬──────────┬─────────────┐
│ All  │Unassigned│ Instagram │WhatsApp │ Walk-in  │   Trials    │
│ كل  │  بدون   │ إنستجرام  │واتساب  │  زيارة  │   تجارب     │
│ الليدز│  مندوب │           │         │          │             │
└──────┴─────────┴───────────┴─────────┴──────────┴─────────────┘
```

**لماذا كل تبويب:**
- **All:** نظرة عامة كاملة
- **Unassigned:** ليدز لم يُعيَّن لهم مندوب — يجب توزيعهم سريعاً
- **Instagram/WhatsApp/Walk-in:** لمعرفة أي مصدر يعطي أفضل نتائج
- **Trials:** كل من لديه موعد تجربة — لمتابعتهم بعد التجربة
- **Follow Up:** كل من يحتاج متابعة الآن

---

## فلاتر البحث وكيفية استخدامها:

| الفلتر | لماذا تستخدمه | مثال |
|---|---|---|
| **بحث بالاسم/هاتف** | إيجاد ليد بسرعة | "أحمد" أو "01012..." |
| **Branch** | عرض ليدز فرع معيّن | COMPLEX فقط |
| **Stage** | ليدز في مرحلة معيّنة | Trial فقط |
| **Interest** | حسب مستوى الاهتمام | Interested فقط |
| **Assigned To** | ليدز مندوب معيّن | "كريم" فقط |
| **Sort by Score** | ترتيب حسب الأولوية | الأعلى احتمالاً أولاً |

```

## لماذا موجود؟
لإدارة كل الأعضاء الحاليين: متابعة اشتراكاتهم، جلساتهم، تجديداتهم، والتواصل معهم.

## متى تستخدمه؟
- لتسجيل عضو جديد
- لمتابعة الاشتراكات القريبة الانتهاء
- للبحث عن عضو محدد
- لمراجعة سجل عضو

---

## حالات العضو وما تعنيها:

```
┌──────────────────────────────────────────────────────────────────┐
│                    دورة حياة العضو                               │
│                                                                  │
│  Lead ──► Active ──► Nearly Expired ──► Expired                  │
│   👤      ✅ نشط    ⚠️ ينتهي < 30 يوم   ❌ منتهي                  │
│                                                                  │
│  أو: Active ──► Hold (متوقف مؤقتاً)                              │
└──────────────────────────────────────────────────────────────────┘
```

| الحالة | المعنى | الإجراء المطلوب |
|---|---|---|
| **Active** ✅ | اشتراك نشط وجلسات متبقية | لا شيء |
| **Nearly Expired** ⚠️ | ينتهي خلال 30 يوم | تواصل لتجديد |
| **Expired** ❌ | انتهى الاشتراك | تواصل عاجل للتجديد |
| **Hold** ⏸️ | متوقف مؤقتاً | انتظر قراره |
| **Lead** 👤 | لم يشترك بعد | تابع لتحويله |

```
┌─────────┬──────────┬──────────┐
│ Active  │   Hold   │ Expired  │

└─────────┴──────────┴──────────┘
```

**الفلاتر الإضافية:**
- بحث بالاسم / هاتف / Member ID

## Renewal Pipeline (خط أنابيب التجديد):

**أين:** زر "Renewal Pipeline" في أعلى صفحة Members
**لماذا:** أداة ذكية تجمع كل من يحتاج تجديد في مكان واحد مرتبين حسب الأولوية
**متى:** كل يوم — خاصةً في بداية ومنتصف الشهر

```
مخطط Renewal Pipeline:

┌─────────────────────────────────────────────────────────────────┐
│                      Renewal Pipeline                           │
├──────────────────┬─────────────────┬──────────────────────────┤
│  🔴 Expired       │  🟡 Urgent       │  🔵 Upcoming              │
│  انتهى الاشتراك  │  ≤ 7 أيام       │  8-30 يوم                 │
│                  │                 │                            │
│  [كريم]          │  [سارة]          │  [أحمد]                   │
│  انتهى منذ 5 أيام│  ينتهي بعد 3    │  ينتهي بعد 20 يوم         │
│  [Mark Contacted]│  [Mark Contacted]│  [Mark Contacted]         │
│  [Record Renewal]│  [Record Renewal]│  [Record Renewal]         │
│                  │                 │                            │
│  [علي]           │  [هند]           │  [نور]                    │
│  انتهى منذ يومين│  ينتهي بعد 5    │  ينتهي بعد 15 يوم         │
└──────────────────┴─────────────────┴──────────────────────────┘
```

**"Mark Contacted":** يسجّل أنك تواصلت مع العضو (يرفع تاريخ آخر تواصل)
**"Record Renewal":** يفتح نافذة تسجيل دفعة تجديد مباشرة

                                                               ▼

                                                               │
                                                               ▼

# 6. المدفوعات

## أين؟

## لماذا موجود؟
لتسجيل كل عملية دفع وربطها بالعضو والمندوب والباكدج. هذا أساس حساب الإيراد والعمولات.

## متى تستخدمه؟
- فور استلام أي دفعة (اشتراك جديد أو تجديد)

```

        │
        ▼ (يملأ الاسم والمندوب تلقائياً)
        │

        │
